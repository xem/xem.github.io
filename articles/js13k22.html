Before the jam

Since js13kgames 2021, I've made a few games:

- IT'S TOO RAW!, my entry for GamedevJS 2022 (PLAY / Github)
- MiniWordle, a Wordle clone in 214 bytes (PLAY / Github)

... and tools:

- MiniMidi, to easily extract notes from a MIDI file (DEMO / Github)
- Midi2Array, a more complete tool to extract notes from MIDI files (DEMO / Github)
- W, a WebGL2 micro-framework developed with @stas (DEMO / Github)
- Zipzop, a tool to compress a file with Zip and Zopfli, online (DEMO / Github)
- MiniCode, to easily find the event.code of Keyboard events (DEMO / Github)
- Unicode13, lists and tools for Unicode 13.0 glyphs and emoji (Github)
- Mini GLSL minifier, a micro WebGL 1.0/2.0 shader minifier made with 10 regexes (DEMO / Github)

10 Years already!

I also remarked that in 2022, I've been Code Golfing in JS for 10 years,
so it's a good occasion to celebrate it!

So I've made a list of all the micro games I had created or worked on since 2012,
and took the challenge to put them all in a sigle js13kgames entry.

It took me two weeks to secretly select, fix, resize, clean up and regolf a total of 28 games and 2 emulators (CHIP-8 and NES), and present them in the form of a bok with the source code on the left and the playable demo on the right.
The content is so dense that it required 4 "hard" passes of RoadRoller, Zip, and Zopfli to make the entry fit into 13300 bytes.

The full readme with source and instructions for each game is available on Github.

The less easy part was to create dynamic iframes with sometimes astral Unicode characters in their source code, focus them, make them refreshable, etc. but I finally manage to do everything as elegantly as possible.
