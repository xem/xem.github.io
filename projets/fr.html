Alignement de texte

juin/juillet 2013

Développement d'une solution en pur CSS pour permettre à du texte d'être centré lorsqu'il y en a peu, et le faire s'aligner à gauche quand il y en a beaucoup. (IE6+)

Puis, développement d'une solution plus générique pour permettre à du texte d'être centré dans une zone précise lorsqu'il y en a peu, et étirer cette zone en alignant le texte à gauche quand il y en a beaucoup (IE8+).

Cas simple: du texte "centré-gauche"

Code HTML:

<div class=align-outer>
	<div class=align-inner>
		Texte texte texte
	</div>
</div>

Code CSS:

.align-outer {
	text-align: center;
}
.align-inner {
	margin: 0 auto;
	text-align: left;
	display: inline-block;
	*display: inline; *zoom: 1; /* IE < 8 */
}

Démo:
(le conteneur autour du texte est représenté par des pointillés rouges, pour voir son étirement)

Texte texte texte texte texte




Cas plus complexe: texte "centré-étirable-gauche"

Code HTML:

<div class=align-stretch-outer>
	<div class=align-stretch-inner-1>
		<div class=align-stretch-inner-2>
			Texte texte
		</div>
	</div>
</div>

Code CSS:

.align-stretch-outer {
	text-align: left;
}
.align-stretch-inner-1 {
  display: inline-block;
  text-align: center;
  min-width: 60%;
}
.align-stretch-inner-2 {
  display: inline-block;
  text-align: left;
}

Démo:
(les conteneurs autour du texte sont représentés par des pointillés rouges, pour voir leur étirement)

Texte texte

NB: l'alignement texte "étiré" est réglé sur "left", mais ça fonctionne aussi avec justify, center ou right via la première règle CSS.

Lightml

mars 2013

Les pages HTML contiennent beaucoup de redondance. Cette expérience vise à compresser le HTML côté serveur (en PHP), et à le décompresser côté client (en JavaScript). Les résultats montrent qu'on peut économiser jusqu'à 46% de bande passante (sans Gzip), et 10% avec Gzip activé.

https://github.com/xem/lightml
